What is logged in the LogBook?

The LogBook really just logs any troubles, interesting observations, and hopefully solutions related to developing the {{ site.brand }} content. To start with, I'll focus on three topics.

LaTeX to MathML

I'm intending to use LaTeX syntax within MarkDown and web scripts to produce the MathML eventually visible on the web pages of {{ site.brand }}. However, I expect this will not always be a gentle ride. Thus, it makes sense to log LaTeX to MathML related issues.

MathML usability

I'm aware that MathML is a standardised markup language. However, what is not standardised is the way different screen readers handle it. I hope to take notes on anything related to MathML usability. One day this might come handy.

Issues

Then there is a general page to track issues. This can be issues with the quality of the creative content, issues with web development, or issues with accessibility. Some of this issue tracking will happen through short posts, and some will happen through more robust methods, such as GitHub's own issue tracking. Yes, I know. Issue is an overused word. Issue-Issue-Issue... Any issues with that?

Good stuff! To check what's been logged so far, have a look at the articles in the side bar.
